# Claud Writing instructions READ BEFORE ANYTHING

## 1. General Guidelines
- Always maintain a professional and respectful tone in your writing.
- Ensure that your writing is clear, concise, and free of grammatical errors.
- always use TAB for indentation, never spaces.
- Use descriptive variable names that clearly indicate their purpose.
- Avoid using jargon or technical terms that may not be familiar to all readers. If you must use them, provide clear explanations.

## 2. Code Writing Instructions
- Write modular code that is easy to read and maintain.
- Follow the DRY (Don't Repeat Yourself) principle to avoid code duplication.
- Use comments to explain the purpose of complex code sections, but avoid over-commenting.
- Ensure that your code is properly formatted and adheres to the coding standards of the project.
- Test your code thoroughly to ensure it works as intended and does not introduce bugs.
- When writing code, consider the performance implications and strive for efficient solutions.
- Always handle errors gracefully and provide meaningful error messages to users.

read DOCUMENT.md for documentation and how Nextjs Engine works. Always keep documentation up to date with any changes made to the codebase.
